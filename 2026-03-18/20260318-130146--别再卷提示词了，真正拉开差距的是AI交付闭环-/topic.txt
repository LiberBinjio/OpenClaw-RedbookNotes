## Viral References
> 只拆高传播方法，不复写原句；只借结构，不照搬表达。

### Reference Pattern A｜认知反转型
- 常见爆点：很多人还在比谁更会写 prompt，真正拉开差距的已经是能不能把一整段流程接起来。
- 可借用模式：先否定旧共识，再抛出新分水岭。
- 本轮映射：把焦点从“会不会问”切到“能不能把 skill 接成 workflow 并交付结果”。

### Reference Pattern B｜结果交付型
- 常见爆点：用户真正想拿走的不是一个聪明回答，而是“它能不能替我做完一段工作”。
- 可借用模式：从“回答质量”改写成“返工更少、结果更稳、动作可复用”。
- 本轮映射：适合内容团队、创作者、开发协作三类场景。

### Reference Pattern C｜前后对比型
- 常见爆点：以前每次从 0 开始提问，现在把动作拆成 skill 节点后，流程更稳。
- 可借用模式：用“反复返工”对比“流程复用”。
- 本轮映射：适合写选题反复改、写稿来回修、审稿缺校验这类连续场景。

### Reference Pattern D｜团队资产型
- 常见爆点：一个人会用 AI，不等于一支团队已经拥有可复制的 AI 生产力。
- 可借用模式：用“个人技巧”对比“团队可交接流程”。
- 本轮映射：强调 skill / workflow 更容易沉淀成团队资产，而不是个人灵感。

## Rewrite Angle
- 不把主题写成“哪个工具又更新了什么”，而写成“为什么 AI 技能内容开始从 prompt 升级到 workflow”。
- 不把重点放在模型有多聪明，而放在 `skill -> workflow -> 交付` 这条升级路径。
- 用 `OpenClaw` 承接“能力开放与可落地插件化”，用 `Copilot` 承接“大众熟知的单点辅助参照”，再把 `AI skill / workflow skill` 收束成“能按步骤把事情做完的 AI 能力包”。
- 内容视角偏向创作者、内容团队、开发协作都能代入的高频场景，而不是新闻播报。

## Hook Breakdown
1. **认知反差钩子**
   - 句式方向：`很多人还在卷哪句 prompt 更神，但真正开始拉开差距的，已经是谁能把一整段流程稳定跑完。`
   - 作用：先制造旧认知失效的反差。
2. **痛点代入钩子**
   - 句式方向：`AI 回一句话很快，但一旦进入选题、写稿、审稿这些连续动作，很多流程还是会断在中间。`
   - 作用：把“单次惊艳、连续失稳”的问题说出来。
3. **结果承诺钩子**
   - 句式方向：`真正更值钱的，不是多会问，而是把高频动作封装成 skill，再接成可以交付的 workflow。`
   - 作用：快速给出新结论。
4. **团队升级钩子**
   - 句式方向：`一个人会用 AI，只能叫技巧；一支团队能复用同一套流程，才叫生产力。`
   - 作用：把话题从个人灵感拉到团队资产。
5. **低门槛动作钩子**
   - 句式方向：`先别急着找下一句神 prompt，先把你最高频的 3 个动作封装成 skill。`
   - 作用：让读者觉得今天就能执行。

## Viral Copy Breakdown
### 1. Hook Pattern
- 高传播内容常用结构：`你以为重点还是 A，其实真正拉开差距的已经是 B`。
- 本轮保留：`prompt/会不会问` 对比 `workflow/能不能做完` 的认知反差。
- 本轮改写：把旧关注点写成“会不会写一句提示词”，把新关注点写成“能不能稳定交付一整段流程”。
- 本轮禁抄：任何现成爆文的反转标题原句。

### 2. Structure Pattern
- 推荐结构：
  1) 先指出旧理解正在失效
  2) 再点出新的内容分水岭
  3) 用 2-3 个连续场景举例
  4) 最后给一个可以立刻执行的微动作
- Draft 可直接套成：反差开头 → 高频场景 → skill/workflow 解释 → 可执行建议。
- 本轮禁抄：资讯原文段落顺序、导语句、总结金句。

### 3. Conflict Point
- 表层冲突：普通用户还在追更强模型、更神 prompt。
- 深层冲突：更高阶的创作者和团队已经开始沉淀 skill 库并做 workflow 编排。
- 这个冲突值得保留，因为天然适合评论区讨论与身份代入。

### 4. Promise
- 给读者的承诺不该是“立刻爆”，而应是：知道下一阶段更该练什么能力。
- 更可信的承诺是：少重复提问、减少返工、让流程可复用、让团队可协作、让结果能交付。
