## Viral References
> 只拆高传播方法，不复写原句；只借结构，不照搬表达。

### Reference Pattern A｜认知反转型
- 常见爆点：很多人还在写某个 AI 提示词多强，真正更容易被收藏的已经是“这套流程怎么直接跑完”。
- 可借用模式：先否定旧共识，再抛出新的内容分水岭。
- 本轮映射：把焦点从“提示词怎么写”切到“skill 怎么接成 workflow，并最终完成交付”。

### Reference Pattern B｜整包交付型
- 常见爆点：用户真正想带走的不是一个零散功能，而是“装上以后我到底能直接完成什么”。
- 可借用模式：从“功能介绍”改写成“场景结果、动作顺序、可复用路径”。
- 本轮映射：适合写选题、改写、复审、归档等动作如何被串成完整内容链路。

### Reference Pattern C｜前后对比型
- 常见爆点：以前到处找 prompt、教程、工具；现在直接要一条已经被封装好的工作流路径。
- 可借用模式：用“零散技巧很散”对比“流程化能力可直接复用”。
- 本轮映射：适合创作者、开发者、内容团队三个角色共用。

### Reference Pattern D｜团队资产型
- 常见爆点：一个人会用 AI 不等于团队已经拥有稳定交付能力。
- 可借用模式：用“个人会用工具”对比“团队有可交接的流程资产”。
- 本轮映射：强调真正有差距的不是谁 prompt 更长，而是谁能稳定完成选题、改写、复审、归档与复用。

## Hook Breakdown

   - 句式方向：`很多人还在卷提示词，但真正开始拉开差距的，已经是谁能把 AI 流程稳定交付出来。`
   - 作用：先制造旧认知失效的反差。
2. **痛点代入钩子**
   - 句式方向：`会问一句 prompt 不难，难的是这条内容线索怎么搜、怎么改、怎么审、怎么留下来继续复用。`
   - 作用：把“会提问”和“会交付”之间的差异说出来。
3. **结果承诺钩子**
   - 句式方向：`真正更值钱的，不是又多了一个新工具，而是你能不能拿到一条今天就能跑起来的完整链路。`

4. **团队升级钩子**
   - 句式方向：`一个人会用 AI 只是技巧，一支团队能共用同一套流程才叫生产力。`
   - 作用：把话题从个人灵感拉到团队资产。
5. **低门槛动作钩子**
   - 句式方向：`先别急着再抄一句 prompt，先看看你最高频的 3 个内容动作能不能接成一条固定流程。`
   - 作用：让读者觉得今天就能执行。

### 2. Structure Pattern
- 推荐结构：
  1) 先指出旧写法正在失效
  2) 再点出新的内容分水岭
  3) 用 2-3 组公开线索举例

- Draft 可直接套成：反差开头 → 场景痛点 → Newrank 邻近线索 → 为什么流程内容更容易被收藏 → 可执行建议。
- 本轮禁抄：资讯原文段落顺序、导语句、总结金句。

### 3. Conflict Point
- 表层冲突：很多内容还停在“教一句提示词”或“讲一个工具功能”。
- 深层冲突：更高阶的创作者和团队已经开始关注 `skill 如何组合成 workflow，workflow 如何稳定交付`。
- 这个冲突值得保留，因为天然适合评论区讨论与身份代入。
