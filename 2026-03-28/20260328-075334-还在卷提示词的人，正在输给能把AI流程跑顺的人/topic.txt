## Viral References
> 只拆高传播方法，不复写原句；只借结构，不照搬表达。

### Reference Pattern A｜认知反转型
- 常见爆点：很多人还在写某个 AI 工具多强，真正更容易被收藏的已经是“这套技能怎么接成流程”。
- 可借用模式：先否定旧共识，再抛出新的内容分水岭。
- 本轮映射：把焦点从“OpenClaw 好不好用”切到“OpenClaw 的 skills 能不能拼成一条完整交付链路”。

### Reference Pattern B｜整包交付型
- 常见爆点：用户真正想带走的不是一个零散功能，而是“装完以后我能直接干什么”。
- 可借用模式：从“功能介绍”改写成“场景结果、动作顺序、可复用路径”。
- 本轮映射：适合写 SkillHub、5 skills workflow、托管部署这类“装上就能跑”的场景。

### Reference Pattern C｜前后对比型
- 常见爆点：以前到处找工具、脚本、教程；现在直接拿一组现成 skills 或 workflow 开始跑。
- 可借用模式：用“自己拼装很散”对比“技能组合后可直接复用”。
- 本轮映射：适合创作者、开发者、内容团队三个角色共用。

### Reference Pattern D｜团队资产型
- 常见爆点：一个人会装 OpenClaw，不等于团队已经拥有稳定的 AI 生产力。
- 可借用模式：用“个人会用工具”对比“团队有可交接的技能路径”。
- 本轮映射：强调 skill/workflow 更容易沉淀成团队资产，而不是一次性灵感。

4. **团队升级钩子**

   - 作用：让读者觉得今天就能执行。

## Viral Copy Breakdown

### 2. Structure Pattern
- 推荐结构：
  1) 先指出旧写法正在失效
  2) 再点出新的内容分水岭
  3) 用 2-3 组公开线索举例

- 本轮禁抄：资讯原文段落顺序、导语句、总结金句。

### 3. Conflict Point
- 表层冲突：普通用户还在看单个工具测评。
- 深层冲突：更高阶的创作者和团队已经开始关注技能市场、技能组合与流程复用。
- 这个冲突值得保留，因为天然适合评论区讨论与身份代入。

### 4. Promise
- 给读者的承诺不该是“立刻爆”，而应是：知道下一阶段更该写什么内容、更该沉淀什么能力。
