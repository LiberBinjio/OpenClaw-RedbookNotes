### Reference Pattern A：从功能介绍转向结果交付
- 常见爆点：大家已经看腻了“它有什么功能”，更愿意点开“它到底帮我做完了什么”。
- 本轮映射：把 `AI skill`、`workflow skill` 统一转译为“不是展示功能，而是展示一整段任务如何落地”。

### Reference Pattern B：从会提问转向会执行
- 常见爆点：过去内容在教用户怎么问，现在更容易保存的是怎么连续完成。
- 本轮映射：承接“提示词越来越不够用”的用户疲劳感，转写为“能不能把事情稳定跑完”。

### Reference Pattern C：从工具名转向流程感
- 常见爆点：单个工具名很快被替代，但完整流程更容易被反复复用。
- 本轮映射：即使 `openclaw`、`copilot`、`workflow skill` 当前缺少公开热度细项，也可以围绕“流程资产”切入，而不是围绕“单个名字”硬做资讯解读。

### Reference Pattern D：从热词追逐转向可带走价值
- 常见爆点：真正高收藏的内容，不是告诉你什么火，而是告诉你今天能拿走什么。

## Viral Copy Breakdown
### 1. Hook Pattern
- 高传播内容常用 `你以为大家关心 A，其实大家更想要 B` 的反差结构。
- 本轮保留：`会不会用工具` vs `能不能交付结果`。

- 本轮禁抄：任何现成爆文的完整反转标题原句。

### 2. Structure Pattern
- 常见结构：
  1) 先说旧内容为什么越来越难被收藏
  2) 再点出用户真正想带走的东西
  3) 用 2-3 个工作场景承接
  4) 最后给出新的写法模板
- Draft 可直接套成：旧表达失效 → 用户需求升级 → 场景拆解 → 新内容模板。
- 本轮禁抄：功能测评类爆文的原段落结构和原句。

### 3. Conflict Point
- 表面冲突：AI 会不会用。
- 深层冲突：你是在介绍工具，还是在交付结果。
- 本轮保留：冲突放在“展示能力”与“完成任务”之间。
- 本轮改写：用选题、写稿、整理资料、复盘归档这些工作流场景承接冲突。

### 4. Promise
- 用户真正想听到的承诺不是“更高级”，而是“更省步骤、更稳、更能复用”。
- 本轮 Promise：当你的内容从“这个 AI 会什么”变成“这套路径怎么帮你做完一件事”，用户更容易判断这条内容值不值得收藏。
- 本轮改写：承诺只落在理解、执行、复用，不夸成必爆。
- 本轮禁抄：`照着发就爆`、`无脑抄作业` 等夸张承诺。
